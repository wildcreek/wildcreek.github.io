PJSIP2.7 Android版本编译及构建pjsua2和pjsua

Oct 9, 2017

本人使用的环境：Ubuntu14.0.4 （DigitalOcean 虚拟主机，笔者使用移动宽带选择Singapore节点80ms 左右延迟，推荐链接https://m.do.co/c/1399118d3ab5）

笔者按照PJSIP Android版本的官方编译说明（https://trac.pjsip.org/repos/wiki/Getting-Started/Android）操作，遇到了部分问题。特别是如Android SDK 依赖问题，按照说明（https://developer.android.com/ndk/downloads/index.html）获取的sdk仅为Android Studio IDE版本或者单独的 sdkmanager 版本（sdkmanager需要通过命令行手动下载其他sdk工具和软件包），为环境配置增加过多复杂度和不便，现参照WebRTC项目使用相同的Android SDK 和NDK依赖。

第四步：PJSIP2.7源码编译

cd /root/pjproject-2.7
export ANDROID_NDK_ROOT=/root/webrtc-build-scripts/android/webrtc/src/third_party/android_tools/ndk
TARGET_ABI=armeabi-v7a APP_PLATFORM=android-20 ./configure-android
make dep && make clean && make

注：

  1. ./configure-android 默认构建armeabi 库，如需构建其他ABI如arm64-v8a, armeabi-v7a, x86，可使用类似命令：TARGET_ABI=armeabi-v7a ./configure-android –use-ndk-cflags
  2. 设置APP_PLATFORM=android-20以规避该问题：dlopen failed: cannot locate symbol “rand” referenced by “libpjsua2.so”，请参见https://stackoverflow.com/questions/27338318/cannot-load-library-reloc-library1285-cannot-locate-rand
